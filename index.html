ğŸ” Searching & Sorting Algorithms

â–¼

Searching Algorithms

1. Linear Search

  • Concept: Sequentially checks each element
  • Time Complexity: Best: O(1), Worst/Average: O(n)
for (int i = 0; i < arr.length; i++) {
  if (arr[i] == key) return i;
}

2. Binary Search

  • Concept: Divide and conquer (for sorted arrays only)
  • Time Complexity: Best: O(1), Worst/Average: O(log n)
while (low <= high) {
  int mid = (low + high) / 2;
  if (arr[mid] == key)
    return mid;
  else if (arr[mid] < key)
    low = mid + 1;
  else
    high = mid - 1;
}

Complexity Description Example
O(1) Constant time Accessing an element
O(log n) Logarithmic time Binary search
O(n) Linear time Linear search
O(nÂ²) Quadratic time Bubble sort
O(n log n) Efficient for large datasets Quick sort

Sorting Algorithms

1. Bubble Sort

  • Compares and swaps adjacent elements
  • Complexity: O(nÂ²)
for (i = 0; i < n - 1; i++)
  for (j = 0; j < n - i - 1; j++)
    if (arr[j] > arr[j+1])
      swap(arr[j], arr[j+1]);

2. Insertion Sort

  • Builds a sorted array one element at a time
  • Complexity: O(nÂ²)
for (i = 1; i < n; i++) {
  key = arr[i];
  j = i - 1;
  while (j >= 0 && arr[j] > key)
    arr[j+1] = arr[j--];
  arr[j+1] = key;
}

3. Selection Sort

  • Repeatedly finds the smallest element and moves it to the sorted portion
  • Complexity: O(nÂ²)
for (i = 0; i < n - 1; i++) {
  min = i;
  for (j = i + 1; j < n; j++)
    if (arr[j] < arr[min])
      min = j;
  swap(arr[min], arr[i]);
}
